About Pretty Good Primitives

This is a project for the Cryptography course at NYU Tandon School of Engineering.

Made by Eric Lin, Charlie Thomas and Kyle Timmermans.

Our project will focus on the digital signature functions that we have been presented in class. Our questionnaire asks users to weight between speed, collision resistance, and reversal hardness. We cover the following functions taught in class:

  • RSA
  • Schnorr
  • DSA
  • ECDSA
  • Lamport
  • Gottesman-Chuang scheme

We hope that this would help you make an informed decision in your application!

Compare Digital Signature Schemes

We have included an at-a-glance chart to compare and contrast the different signature schemes.

Runtime and speed complexity is provided by OpenSSL implementations along with their recommended key sizes.

Signature Protocol Hardness Assumption Recommended Minimum Keysize Sign/s Verify/s Quantum Proof Advantage Disadvantage
Hashed RSA RSA + Random Oracle 2048 bits 1900 66896 Easy to implement, withstood the test against time so far Slow data transfer, relies on third party to distribute and verify public keys
DSA Discrete Log over Zp 2048 bits 5621 6366 Signature size is small, efficient to store Verification requires computation time, data is not encrypted
ECDSA Discrete Log over EC 256 bits 61117 20498 Very fast computation for sign and verification compared to RSA and DSA Newer scheme, not widely used, NIST curve controversy
Schnorr Discrete Log 256 bits 32263* 75* Fast signature generation for multi-party setups Verification takes significant computation time
Lamport Signature SHA 256 512 x 256 bits 204* 1865* More secure than tranditional schemes in the face of quantum computing Very large signature sizes, many hash computation needed, one time use
Gottesman-Chuang scheme Unconditional security O(2N log d) - d being message space and N is the codeword size Efficient* Efficient* Efficient and error correction built in Requires quantum computer and qubits
