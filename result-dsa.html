About the DSA Scheme


Explanation

Digital Signature Algorithm (DSA) is part of the Digital Signature Standard (DSS) published by the Federal Information Processing Standard (FIPS). This algorithm generates digital signatures with the help of Secure Hash Algorithms (SHA) for the authentication of electronic documents. Note that DSS only provides the digital signature function and does not do any of the key exchanges and encryption.

Sender Approach:

In a typical DSS implementation, the sender side of the signature function needs the following:

  • The hash code of the message
  • Randomly generated 'k'
  • Private key of the sender

Receiver/Verifier Approach:

Once the sender has sent the message to the receiver, then the verification happens:

  • Retrieve sender public key
  • Verify the message signature based on the public key
